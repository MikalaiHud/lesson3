Описать иерархию классов в рамках произвольной бизнес-модели (автопарк, фототехника, тарифные планы и т.п).
Структура классов должна иметь как минимум один базовый класс и несколько наследников (напр. базовый класс – камеры, наследники – мобильные, компактные, зеркальные).
В базовом классе перечисляются свойства, характерные для всех бизнес-объектов в рассматриваемой области, для классов наследников – специфичные параметры.
Все переменные – свойства классов должны быть приватными (private), для установки/получения значений которых должны быть публичные методы-мутаторы (getters, setters).
Для каждого класса в иерархии должен быть описан конструктор по умолчанию и конструктор со всеми параметрами.

В методе main программы реализовать заполнение объектов классов-наследников пользовательскими данными, простейшие бизнес операции (напр. расчёт средней цены) и вывод данных на консоль
